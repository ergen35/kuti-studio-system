# Plan : Supprimer définitivement `app/lib/api.ts`

## Contexte
Le fichier `app/lib/api.ts` est un wrapper legacy qui fournit des types et fonctions manuelles qui sont maintenant disponibles dans le SDK OpenAPI (`~/lib/backend/`). Ce fichier doit être éliminé pour assurer une source de vérité unique.

## Analyse des dépendances
Actuellement, `api.ts` est utilisé par :
- **3 composants scene** : `CharacterImageSelector.tsx`, `SceneMangaGallery.tsx`, `SceneGenerationModal.tsx`
- **8 composants characters/story** : `CharacterCard.tsx`, `CharacterCardGrid.tsx`, `CharacterDetailModal.tsx`, `CharacterImageGallery.tsx`, `CharacterImageGenerator.tsx`, `GenerationPanelGrid.tsx`, `ImageLightbox.tsx`, puis `StoryBreadcrumb.tsx`, `TomeCard.tsx`, `TomeCardGrid.tsx`
- **3 routes** : `character.tsx`, `scene.tsx`, `chapter.tsx`

## Fonctionnalités à migrer

| Fonctionnalité | Fichier actuel | SDK équivalent | Action requise |
|----------------|----------------|----------------|----------------|
| Types (`Character`, `CharacterImage`, etc.) | `api.ts` type export | `~/lib/backend/types.gen.ts` | Type alias déprécié |
| `api.characterImageUrl()` | `api.ts` | Fonction utilitaire locale | Créer `~/lib/image-urls.ts` |
| `api.generationPanelImageUrl()` | `api.ts` | Fonction utilitaire locale | Créer `~/lib/image-urls.ts` |
| `apiErrorMessage()` | `api.ts` | `~/lib/errors.ts` | Déjà migré |
| `csv()` | `api.ts` | `~/lib/utils.ts` | Déjà migré |
| `ApiError` class | `api.ts` | Error handling du SDK | véifier usage |

## Détail des contrats

### Types à migrer
```typescript
// Dans api.ts (legacy) → types.gen.ts (SDK)
export type Character = { ... } 
  → Pas de type équivalent exact, utiliser GetCharacterResponse[200] ou ListCharactersResponse[200][number]
export type CharacterImage = { ... }
  → ListCharacterImagesResponse[200][number]
export type Tome = { ... }
  → GetStorySummaryResponse[200]['tomes'][number]
export type Chapter = { ... }
  → GetStorySummaryResponse[200]['chapters'][number]
export type Scene = { ... }
  → GetStorySummaryResponse[200]['scenes'][number]
export type CharacterRelation = { ... }
  → GetCharacterResponse[200]['relations'][number]
export type VoiceSample = { ... }
  → GetCharacterResponse[200]['voiceSamples'][number]
export type GenerationPanel = { ... }
  → GetGenerationBoardResponse[200]['panels'][number]
```

### URLs d'images utilitaires
Les fonctions suivantes ne sont pas générées par le SDK mais doivent être préservées :

```typescript
// app/lib/image-urls.ts (nouveau fichier)
import { API_BASE_URL } from "~/lib/errors";

export function characterImageUrl(
  projectId: string, 
  characterId: string, 
  imageId: string
): string {
  return `${API_BASE_URL}/api/projects/${projectId}/characters/${characterId}/images/${imageId}/file`;
}

export function generationPanelImageUrl(
  projectId: string, 
  boardId: string, 
  panelId: string
): string {
  return `${API_BASE_URL}/api/projects/${projectId}/generation/boards/${boardId}/panels/${panelId}/image`;
}
```
