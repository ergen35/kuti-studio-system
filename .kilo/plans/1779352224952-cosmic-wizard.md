# Plan d'implémentation - Home Page Fixes

## Objectifs

1. Retirer l'arrière-plan dynamique `CreativeBackground` de la page d'accueil
2. S'assurer que la création de projet fonctionne de bout en bout
3. S'assurer que la récupération de la liste des projets fonctionne correctement

---

## Analyse actuelle

### Frontend (`home.tsx`)

**Problèmes identifiés :**

1. **CreativeBackground** (ligne 303) utilise `useBackgroundImages()` qui fait des appels API complexes pour récupérer des images de personnages
2. **Création de projet** (lignes 276-298) :
   - Utilise `useCreateProject` hook
   - Envoie `settings_json` dans le body
   - Attend une réponse avec redirection vers `/projects/${project.id}`
3. **Liste des projets** (lignes 148-266) utilise `useProjects()` qui appelle `readProjectsApiProjectsGet`
4. **Métriques mockées** (lignes 91-106) - générées côté client avec `generateProjectMetrics()`

### Backend (`projects/`)

**Structure OK :**
- `GET /api/projects` → `listProjects()`
- `POST /api/projects` → `createProject()`
- DTO avec `settingsJson` (camelCase dans le code)

**Point de friction potentiel :**
- Frontend envoie `settings_json` (snake_case)
- Backend attend `settingsJson` (camelCase)
- Le SDK Hey API génère des schémas avec `settings_json` (snake_case)

## Notes techniques

### Format des réponses API

**Backend (listProjects) :**
```typescript
return projects.map(serializeProject);
// Retourne : ProjectResponse[]
```

**Hey API génère :**
- Les noms de propriétés suivent le schéma OpenAPI
- Si le schéma utilise `settings_json` (snake_case), le SDK utilisera `settings_json`
- La conversion camelCase/snake_case dépend de la config OpenAPI

### Points de vigilance

1. **TypeScript** : Les `as unknown as Project` suggèrent des incompatibilités de types à résoudre
2. **Métriques** : Actuellement mockées côté client avec une fonction basée sur le hash de l'ID
3. **Cache** : Après création, `invalidateQueries` est appelé pour rafraîchir la liste
