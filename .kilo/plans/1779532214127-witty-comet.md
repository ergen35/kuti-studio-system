# Plan: Suppression de projet avec confirmation et job Inngest

## Objectif
Ajouter un bouton de suppression dans les paramètres du projet avec confirmation par saisie du nom, puis exécution via un job Inngest pour supprimer les données et fichiers associés.

## Architecture

### 1. Backend - Route API (kuti-backend/src/modules/projects/)

**Fichier: `index.ts`**
- Ajouter route `DELETE /api/projects/:projectId`
- Body: `deleteProjectBodySchema` avec confirmation du nom
- Response: `{ jobId: string, status: "pending" }`
- OperationId: `deleteProject`

**Fichier: `dto/index.ts`**
- Ajouter `deleteProjectBodySchema` avec champ `confirmedName` (string)
- Ajouter type `DeleteProjectBody`

**Fichier: `controller.ts`**
- Ajouter fonction `deleteProject(projectId, confirmedName)`:
  - Vérifier que le projet existe
  - Vérifier que `confirmedName === project.name`
  - Créer un job Inngest (ou un record de traitement)
  - Envoyer l'événement Inngest `kuti/delete-project`
  - Retourner le jobId

### 2. Backend - Workflow Inngest (kuti-backend/src/lib/inngest/)

**Fichier: `delete-project.ts` (nouveau)**
- Créer la fonction `deleteProjectFunction`
- Événement: `kuti/delete-project`
- Steps:
  1. `fetch-project`: Récupérer le projet et toutes ses relations
  2. `delete-files`: Supprimer le répertoire `rootPath` et sous-dossiers (assets, generation, exports)
  3. `delete-database`: Supprimer le projet en base (cascade vers characters, tomes, chapters, scenes, assets, generation jobs, etc.)
  4. `finalize`: Marquer comme complété
- Gestion des erreurs: logger et propager

**Fichier: `index.ts`**
- Exporter `deleteProjectFunction` et `sendDeleteProjectEvent`
- Ajouter à `inngestFunctions`
- Ajouter le type `DeleteProjectEvent` dans les événements

### 3. Frontend - Page Settings (kuti-frontend/app/routes/settings.tsx)

**Ajouter les imports:**
- `Dialog` (à créer ou importer de ~/components/ui si existe)
- `useNavigate` de react-router

**Ajouter le state:**
- `isDeleteDialogOpen`: boolean
- `deleteConfirmationInput`: string
- `deleteMutation`: useMutation pour appeler deleteProject

**Ajouter la section "Zone de danger" :**
- Panel avec bordure rouge/danger
- Titre: "Supprimer le projet"
- Description: "Cette action est irréversible..."
- Bouton "Supprimer le projet" (variant danger)

**Créer le Dialog de confirmation:**
- Titre: "Confirmer la suppression"
- Message: "Pour confirmer, tapez le nom du projet : {projectName}"
- Input text pour saisir le nom
- Bouton "Annuler" (ferme le dialog)
- Bouton "Confirmer la suppression" (disabled tant que input !== projectName)
- Au clic: lancer la mutation et rediriger

**Mutation:**
- Appeler `deleteProjectMutation` avec projectId et confirmedName
- OnSuccess: redirect vers `/` (home)
- OnError: afficher l'erreur

### 4. Frontend - SDK API (kuti-frontend/app/lib/backend/)

**Après modification du backend:**
- Exécuter `yarn openapi-ts` pour régénérer le SDK
- Vérifier que `deleteProjectMutation` et `deleteProjectBodySchema` sont créés

### 5. i18n - Traductions (kuti-frontend/app/locales/)
