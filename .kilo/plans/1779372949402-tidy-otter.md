# Plan : Supprimer use-api.ts, utiliser directement le SDK

## Contexte
Le fichier `app/hooks/use-api.ts` crée des wrappers inutiles autour des hooks TanStack Query générés par @hey-api/openapi-ts. L'utilisateur veut utiliser directement le SDK sans couche d'indirection.

## Problème identifié
Les imports dans `use-api.ts` utilisent des noms incorrects qui ne correspondent pas aux exports réels du SDK :
- ❌ `readProjectsApiProjectsGetOptions` → ✅ `listProjectsOptions`
- ❌ `readProjectApiProjectsProjectIdGetOptions` → ✅ `getProjectOptions`
- ❌ `createProjectRouteApiProjectsPostMutation` → ✅ `createProjectMutation`

## Changements par route

### routes/project.tsx
```typescript
// AVANT
import { useProject, useCharacters, ... } from "~/hooks/use-api";
const project = useProject(projectId);

// APRÈS
import { getProjectOptions, listCharactersOptions, ... } from "~/lib/backend/@tanstack/react-query.gen";
import { useQuery } from "@tanstack/react-query";
const project = useQuery({ ...getProjectOptions({ path: { project_id: projectId } }), enabled: !!projectId });
```

### routes/home.tsx
```typescript
// AVANT
import { useProjects, useCreateProject } from "~/hooks/use-api";
const projects = useProjects();
const create = useCreateProject({ mutation: { onSuccess: ... } });

// APRÈS
import { listProjectsOptions, createProjectMutation } from "~/lib/backend/@tanstack/react-query.gen";
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
const projects = useQuery(listProjectsOptions());
const queryClient = useQueryClient();
const create = useMutation({
  ...createProjectMutation(),
  onSuccess: (data) => {
    queryClient.invalidateQueries({ queryKey: ["listProjects"] });
    // ...
  }
});
```
