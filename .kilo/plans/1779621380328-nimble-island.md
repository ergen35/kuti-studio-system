# Plan: Boutons Annuler et Relancer pour les Tâches

## Vue d'ensemble

Implémenter les fonctionnalités **Annuler** (cancel) et **Relancer** (relaunch) pour les jobs de génération via Inngest. Ces actions seront disponibles depuis la boîte de dialogue de détails d'une tâche.

## Architecture

```
Frontend (React)                    Backend (Elysia)                  Inngest
     |                                    |                              |
     | POST /jobs/:jobId/cancel          |                              |
     |---------------------------------->|                              |
     |                                    | send("kuti/job.cancel")      |
     |                                    |----------------------------->|
     |                                    |                              |-- [cancel running job]
     |                                    |                              |
     | POST /jobs/:jobId/relaunch        |                              |
     |---------------------------------->|                              |
     |                                    | send("kuti/job.relaunch")    |
     |                                    |----------------------------->|
     |                                    |                              |-- [clone & restart job]
```

#### 1.2 Implémentation des fonctions controller dans `controller.ts`

**`cancelGenerationJob(projectId, jobId)`:**
- Vérifier que le job existe et appartient au projet
- Vérifier que le job est en status "running" ou "pending"
- Envoyer un événement Inngest `kuti/job.cancel` avec le jobId
- Retourner `{ success: true, message: "Cancel request sent" }`

**`relaunchGenerationJob(projectId, jobId)`:**
- Vérifier que le job existe et appartient au projet
- Vérifier que le job est en status "ready", "validated", "failed" ou "pending"
- Créer un nouveau job basé sur les données du job existant:
  - Copier: sourceKind, sourceId, strategy, modelKey, entrypoint, title, prompt, summary
  - Nouveau ID, status: "pending", progress: 0
- Sauvegarder le nouveau job en base
- Envoyer un événement Inngest `kuti/job.relaunch` avec:
  - `newJobId`, `originalJobId`, `projectId`, `sourceKind`, `sourceId`
- Retourner le nouveau job créé

### Phase 2: Backend - Fonctions Inngest (kuti-backend/src/lib/inngest)

#### 2.1 Nouveau fichier: `job-control.ts`

Créer un fichier pour gérer les événements de contrôle des jobs:

```typescript
import { inngest } from "./client";
import db from "@lib/db";

// Event type: kuti/job.cancel
export const cancelJobFunction = inngest.createFunction(
  { id: "cancel-generation-job" },
  { event: "kuti/job.cancel" },
  async ({ event, step }) => {
    const { jobId, projectId } = event.data;
    
    // Note: Inngest ne permet pas de "vraiment" cancel un job en cours
    // On va plutôt:
    // 1. Marquer le job comme "failed" avec raison "cancelled"
    // 2. Si c'est une génération en cours, attempt d'arrêter le process
    
    await step.run("mark-job-cancelled", async () => {
      await db.generationJob.update({
        where: { id: jobId },
        data: { 
          status: "failed",
          errorMessage: "Cancelled by user",
          failedAt: new Date()
        }
      });
    });
    
    return { success: true, jobId, action: "cancelled" };
  }
);

// Event type: kuti/job.relaunch
export const relaunchJobFunction = inngest.createFunction(
  { id: "relaunch-generation-job" },
  { event: "kuti/job.relaunch" },
  async ({ event, step }) => {
    const { newJobId, projectId, sourceKind, sourceId } = event.data;
    
    // Le job est déjà créé en pending, on déclenche juste l'exécution
    // selon le type de génération
    
    if (sourceKind === "scene") {
      // Déclencher la génération de scene manga
      await step.run("execute-scene-generation", async () => {
        // Logique existante de génération
      });
    } else if (sourceKind === "character") {
      // Déclencher la génération d'image de personnage
      await step.run("execute-character-generation", async () => {
        // Logique existante
      });
    }
    // etc...
    
    return { success: true, jobId: newJobId, action: "relaunched" };
  }
);

// Send helpers
export async function sendCancelJobEvent(data: { jobId: string; projectId: string }): Promise<void> {
  await inngest.send({
    name: "kuti/job.cancel",
    data,
  });
}

export async function sendRelaunchJobEvent(data: { 
  newJobId: string; 
  originalJobId: string;
  projectId: string;
  sourceKind: string;
  sourceId: string;
}): Promise<void> {
  await inngest.send({
    name: "kuti/job.relaunch",
    data,
  });
}
```

```typescript
export function TaskDetailDialog({
  task,
  projectId,
  isOpen,
  onClose,

  const queryClient = useQueryClient();
  
  // Conditions d'affichage des boutons
  const canCancel = task.status === "running" || task.status === "pending";
  const canRelaunch = task.status === "ready" || task.status === "validated" || task.status === "failed";
  
  // Mutation Annuler
  const cancelMutation = useMutation({
    ...cancelGenerationJobMutation(),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["listGenerationJobs"] });
      // Optionnel: fermer le dialog ou afficher un toast
    },
  });
  
  // Mutation Relancer
  const relaunchMutation = useMutation({
    ...relaunchGenerationJobMutation(),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["listGenerationJobs"] });
    },
  });
  
  const handleCancel = () => {
    cancelMutation.mutate({ 
      path: { projectId, jobId: task.id } 
    });
  };
  
  const handleRelaunch = () => {
    relaunchMutation.mutate({ 
      path: { projectId, jobId: task.id } 
    });
  };
  
  // ... reste du composant ...
}
```

## Dépendance entre jobs (cancel)

Pour gérer proprement l'annulation d'un job parent avec des enfants:

```typescript
// Dans cancelJobFunction
await step.run("cancel-parent-and-children", async () => {
  // Annuler le job parent
  await db.generationJob.update({
    where: { id: jobId },
    data: { status: "failed", errorMessage: "Cancelled by user" }
  });
  
  // Annuler tous les jobs enfants associés (si présents dans metadataJson)
  const job = await db.generationJob.findUnique({ where: { id: jobId } });
  const metadata = job?.metadataJson as { children?: string[] };
  if (metadata?.children?.length) {
    await db.generationJob.updateMany({
      where: { id: { in: metadata.children } },
      data: { status: "failed", errorMessage: "Parent job cancelled" }
    });
  }
});
```
